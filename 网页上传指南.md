# 🌐 网页上传方案（最简单）

如果你觉得命令行复杂，可以直接通过GitHub网页上传文件。

## 方案A：网页上传所有文件

### 步骤1：创建新仓库（如果还没创建）
1. 访问：https://github.com/new
2. 填写：
   - Repository name: `daily_stock_analysis`
   - Description: `股票智能分析系统`
   - 选择：Public
   - **不要**勾选 "Initialize this repository with..."
3. 点击 "Create repository"

### 步骤2：上传文件
1. 进入新创建的仓库
2. 点击 "Add file" → "Upload files"
3. 选择 `/root/.openclaw/workspace/daily_stock_analysis/` 目录下的所有文件
4. 点击 "Commit changes"

### 需要上传的关键文件：
- `.env` - 主配置文件
- `配置说明.md` - 配置文档
- `快速启动.md` - 启动指南
- `GitHub部署指南.md` - 部署说明
- `check_config.py` - 检查脚本
- `test_simple.py` - 测试脚本
- `test_analysis.py` - 分析测试
- 其他所有项目文件

## 方案B：只上传必要文件

如果上传所有文件太麻烦，只上传这些必要文件：

### 核心配置文件：
1. **`.env`** - 包含所有配置
2. **`requirements.txt`** - 依赖列表
3. **`main.py`** - 主程序

### 创建方式：
1. 在GitHub仓库中，点击 "Add file" → "Create new file"
2. 文件名输入：`.env`
3. 复制以下内容：

## 方案C：使用GitHub Desktop

1. **下载GitHub Desktop**：https://desktop.github.com/
2. **克隆仓库**：File → Clone repository → 你的仓库

## 启用GitHub Actions

1. 访问：`https://github.com/CHEN19871231/daily_stock_analysis/actions`
2. 点击 "I understand my workflows, go ahead and enable them"
3. 手动运行：点击 "每日股票分析" → "Run workflow" → 选择 `full`
